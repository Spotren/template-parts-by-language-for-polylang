# Security Policy

## Supported versions

Security fixes are applied to the latest released version of the plugin.

## Reporting a vulnerability

Please do not report security vulnerabilities in public GitHub issues.

Instead, report them privately with:

- A description of the issue
- Affected versions
- Reproduction steps or a proof of concept
- Impact assessment if known

If no dedicated private contact is listed in the repository profile, open a GitHub security advisory or contact the maintainer privately before public disclosure.

## Disclosure policy

After a report is received, the maintainer will review the issue, confirm impact, prepare a fix, and coordinate disclosure when a patch is available.
